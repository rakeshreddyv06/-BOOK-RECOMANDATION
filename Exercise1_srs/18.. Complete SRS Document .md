# Software Requirements Specification (SRS)

## 1. Introduction

### 1.1 Purpose

This document specifies the functional and non-functional requirements of the Book Recommendation System. It serves as a reference for developers, instructors, and stakeholders to understand system behavior, constraints, and expectations.

### 1.2 Scope

The Book Recommendation System is a web-based application that provides personalized book recommendations based on user preferences, reading history, and ratings using machine learning techniques such as collaborative filtering, content-based filtering, and hybrid approaches.

### 1.3 Definitions and Acronyms

* **CF**: Collaborative Filtering
* **CBF**: Content-Based Filtering
* **SRS**: Software Requirements Specification

---

## 2. Overall Description

### 2.1 Product Perspective

The system operates as a standalone web application with a user interface, recommendation engine, and database.

### 2.2 Product Functions

* User registration and login
* Book search and browsing
* Personalized book recommendations
* Book rating and feedback
* Admin management of book catalog

### 2.3 User Characteristics

Users are expected to have basic computer literacy and internet access.

### 2.4 Constraints

* Availability and quality of datasets
* Internet connectivity
* Academic time constraints

---

## 3. Functional Requirements

**FR-01: User Registration and Login**
The system shall allow users to register and log in securely.

**FR-02: Book Search**
The system shall allow users to search books by title, author, or genre.

**FR-03: Personalized Recommendations**
The system shall generate personalized book recommendations based on user preferences and interactions.

**FR-04: Book Rating and Feedback**
The system shall allow users to rate books and provide feedback to improve recommendations.

**FR-05: Admin Book Management**
The system shall allow admins to add, update, and remove book records.

---

## 4. Non-Functional Requirements

**NFR-01: Performance**
The system shall respond to user requests within 2 seconds.

**NFR-02: Security**
The system shall ensure secure authentication and data protection.

**NFR-03: Scalability**
The system shall handle an increasing number of users and books without performance degradation.

**NFR-04: Availability**
The system shall be available during normal operational hours with minimal downtime.

---

## 5. Scope Boundaries

The system focuses only on recommending books using stored datasets. Physical book delivery, payment processing, and integration with commercial e-commerce platforms are outside the scope. The project is intended for educational and research purposes.

---

## 6. Stakeholders

Primary stakeholders include users, project developers, instructors, and the university. Each stakeholder has defined interests and influence over the project outcomes.
